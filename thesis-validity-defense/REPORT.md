# Отчёт по защите темы выпускной квалификационной работы

## 1. Описание темы

**Тема:** Разработка кроссплатформенного мобильного приложения для трекинга достижений с элементами геймификации

Предметной областью данной работы является разработка программного обеспечения в сфере персональной продуктивности с применением принципов геймификации. Суть разработки заключается в создании кроссплатформенного приложения, позволяющего пользователям структурировать свои цели в виде системы достижений, отслеживать прогресс их выполнения и обмениваться наборами достижений с другими пользователями.

Приложение реализуется с использованием технологии Kotlin Multiplatform, что обеспечивает возможность развёртывания на платформах Android, iOS и Web из единой кодовой базы. Ключевой особенностью системы является декомпозиция достижений на отдельные шаги с поддержкой как бинарного (выполнено/не выполнено), так и инкрементального (N из M) прогресса.

## 2. Обоснование актуальности

### 2.1. Теоретические предпосылки

Геймификация — применение игровых механик в неигровых контекстах — представляет собой активно развивающееся направление в области проектирования пользовательского опыта. Исследования последних лет демонстрируют значительный потенциал геймификации для повышения мотивации и вовлечённости пользователей.

### 2.2. Обзор литературы

**Zeng (2024)** в статье *"Exploring the impact of gamification on students' academic performance"*, опубликованной в British Journal of Educational Technology, исследует влияние геймифицированных систем на академическую успеваемость студентов. Результаты исследования свидетельствуют о положительной корреляции между применением игровых элементов и образовательными результатами, что подтверждает эффективность геймификации как инструмента повышения мотивации в контексте достижения целей [1].

Исследование *"Goal-setting And Achievement In Activity Tracking Apps: A Case Study Of MyFitnessPal"* рассматривает механизмы целеполагания в приложениях для отслеживания активности. Авторы анализируют, каким образом визуализация прогресса и система достижений влияют на формирование устойчивых поведенческих паттернов. Выводы работы указывают на критическую важность грамотного дизайна системы целей для долгосрочной вовлечённости пользователей [2].

В работе *"How gamification motivates: An experimental study of the effects of specific game design elements on psychological need satisfaction"* представлены результаты экспериментального исследования влияния конкретных игровых элементов на удовлетворение базовых психологических потребностей (автономии, компетентности, связанности с другими). Исследование обосновывает выбор конкретных игровых механик — в частности, систем достижений и прогресс-баров — как наиболее эффективных для поддержания внутренней мотивации [3].

### 2.3. Практическая актуальность

Анализ существующих решений на рынке мобильных приложений выявил отсутствие качественного продукта, сочетающего следующие характеристики:
- кроссплатформенность с единым пользовательским опытом;
- возможность создания и передачи пользовательских наборов достижений;
- стабильный, корректно работающий интерфейс;
- функционирование в офлайн-режиме.

Таким образом, разработка приложения, устраняющего указанные недостатки существующих решений, представляется актуальной задачей.

## 4. Цель и задачи

### Цель работы

Разработать кроссплатформенное мобильное приложение для трекинга достижений, применяющее принципы геймификации для повышения мотивации пользователей при достижении персональных целей.

### Задачи

1. Провести анализ существующих решений в области трекинга достижений и выявить их функциональные и технические недостатки.

2. Спроектировать архитектуру кроссплатформенного приложения на основе Kotlin Multiplatform с учётом требований масштабируемости и поддерживаемости.

3. Реализовать систему достижений с поддержкой иерархической структуры (пакеты → достижения → шаги) и различных типов прогресса.

4. Разработать механизм обмена пакетами достижений между пользователями посредством уникальных кодов.

5. Интегрировать клиентское приложение с серверным API для хранения и синхронизации данных.

6. Провести тестирование и валидацию пользовательского опыта.

## 5. Анализ аналогов

В ходе исследования были проанализированы три приложения, представленные в Google Play Store, позиционирующиеся как инструменты для создания и трекинга персональных достижений.

### 5.1. Achievements (com.comli.rogssgame.mypersonalachievements)

**Google Play:** https://play.google.com/store/apps/details?id=com.comli.rogssgame.mypersonalachievements

Приложение декларирует функционал создания, редактирования и категоризации достижений по уровням ценности (Bronze, Silver, Gold, Platinum).

**Выявленные недостатки:**
- Критические ошибки в работе интерфейса — значительная часть элементов управления не функционирует.
- Некорректная обработка системных отступов (insets), приводящая к перекрытию контента системными элементами.
- Практическая невозможность использования приложения по назначению.

### 5.2. Achievement Box (hcody.hesham.achivementBox.achivement_box)

**Google Play:** https://play.google.com/store/apps/details?id=hcody.hesham.achivementBox.achivement_box

Приложение с геймифицированным подходом, включающее систему внутренней валюты за выполнение привычек и возможность приобретения «наград».

**Преимущества:**
- Стабильная работа интерфейса с базовыми анимациями.
- Наличие статистики и визуализации прогресса.
- Открытый исходный код.
- Поддержка резервного копирования через Google-сервисы.

**Недостатки:**
- Отсутствие возможности передачи достижений другим пользователям.
- Терминологическая путаница (достижения именуются «привычками»).
- Отсутствие социальных функций и профилей пользователей.
- Ограниченный выбор визуального оформления достижений (только предустановленные иконки).

### 5.3. Achievements (com.mycompany.theachievements)

**Google Play:** https://play.google.com/store/apps/details?id=com.mycompany.theachievements

Приложение с фокусом на социальном взаимодействии и AI-генерации достижений.

**Преимущества:**
- Интеграция генеративного AI для создания достижений.
- Система пользователей и профилей с возможностью взаимной выдачи достижений.
- Настраиваемые визуальные стили достижений.

**Недостатки:**
- Некорректная обработка системных отступов.
- Критические ошибки в цветовой схеме (чёрный текст на чёрном фоне).
- Полная зависимость от сетевого подключения — отсутствие кеширования и офлайн-режима.
- Невозможность самостоятельно подтвердить выполнение взятого достижения.
- Предположительно реализовано как WebView-обёртка или на React Native.

### 5.4. Выводы анализа

Ни одно из рассмотренных решений не обеспечивает совокупности следующих характеристик:
- стабильный, качественно реализованный пользовательский интерфейс;
- возможность создания и передачи пользовательских наборов достижений;
- автономная работа (офлайн-режим);
- кроссплатформенность.

Данные выводы подтверждают целесообразность разработки нового решения, устраняющего выявленные недостатки.

## 6. Техническое задание

### 6.1. Выбор технологического стека

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| Мультиплатформенная основа | Kotlin Multiplatform (KMP) | Единая кодовая база для Android, iOS, Web; нативная производительность; поддержка JetBrains |
| UI-фреймворк | Compose Multiplatform | Декларативный подход; единый UI-код для всех платформ; Material 3 |
| Сетевое взаимодействие | Ktor Client | Нативная поддержка KMP; корутины; типобезопасность |
| Генерация API-клиента | OpenAPI Generator | Автоматическая генерация из спецификации; снижение ошибок интеграции |
| Архитектурный паттерн | MVVM | Разделение ответственности; тестируемость; совместимость с Compose |
| Сериализация | kotlinx.serialization | Нативная поддержка KMP; интеграция с Ktor |

### 6.2. Преимущества выбранного стека

1. **Единая кодовая база.** До 95% кода является общим для всех платформ, что существенно сокращает затраты на разработку и поддержку.

2. **Нативная производительность.** В отличие от гибридных решений (React Native, Flutter), KMP компилируется в нативный код целевой платформы.

3. **Декларативный UI.** Compose Multiplatform обеспечивает современный подход к построению интерфейсов с реактивным обновлением состояния.

4. **Типобезопасность.** Статическая типизация Kotlin в сочетании с генерацией API-клиента из OpenAPI-спецификации минимизирует runtime-ошибки.

5. **Экосистема JetBrains.** Качественная поддержка инструментария (IDE, отладчик, профилировщик).

### 6.3. Недостатки и ограничения

1. **Относительная новизна технологии.** Compose Multiplatform и Kotlin/Wasm для Web-платформы находятся в стадии Beta, что может приводить к нестабильности.

2. **Размер итогового бинарного файла.** KMP-приложения имеют больший размер по сравнению с чисто нативными.

3. **Ограниченность библиотечной экосистемы.** Не все популярные библиотеки поддерживают KMP.

### 6.4. Технические риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Несовместимость OpenAPI Generator с Ktor 3+ | Средняя | Среднее | Ручная корректировка сгенерированного кода; фиксация версий |
| Нестабильность Kotlin/Wasm и Compose для Web | Средняя | Среднее | Мониторинг обновлений; приоритизация Android при критичных багах |

| Изменения в API бэкенда | Средняя | Среднее | Версионирование API; контрактное тестирование |
